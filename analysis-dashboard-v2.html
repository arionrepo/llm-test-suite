Model Specialization Matrix
What this shows: Which model performs best for each knowledge type.
How to read: Green = strong (70%+), Yellow = moderate (50-70%), Orange = weak (30-50%), Red = poor (<30%).
Use this to: Select the right model for your specific task type.
Knowledge Type Performance by Model
What this shows: Performance breakdown for each knowledge type across all tested models.
How to read: Each bar represents one model's average score for that knowledge type.
Use this to: Identify which models handle Factual, Relational, Procedural, Synthesis best.
Speed vs Accuracy Tradeoff
What this shows: Response time (X-axis) vs accuracy score (Y-axis) for each model.
How to read: Top-left = fast AND accurate (ideal). Bottom-right = slow AND inaccurate (avoid).
Use this to: Choose between fast/cheap models vs slow/accurate models based on your needs.
Cost Efficiency (Quality per Token)
What this shows: Quality score achieved per token generated (higher = better value).
Formula: Average Score / Average Completion Tokens
Use this to: Find models that give best results for lowest resource usage.
Model Consistency (Reliability)
What this shows: Standard deviation of scores across all tests (lower = more consistent).
How to read: Low bars = predictable, reliable performance. High bars = unpredictable.
Use this to: Choose models for production where you need consistent results.
