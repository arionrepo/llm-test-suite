Model Specialization Matrix
What this shows: Which model excels at which knowledge type. Green = strong performance, Yellow = moderate, Red = weak. Use this to select the right model for your specific task type.
Knowledge Type Performance by Model
What this shows: Performance breakdown for each knowledge type across all models. Helps identify which models handle Factual, Relational, Procedural, Exact Match, and Synthesis best.
Speed vs Accuracy Tradeoff
What this shows: Response time vs accuracy score for each model. Points in top-left = fast AND accurate (ideal). Bottom-right = slow AND inaccurate (avoid). Helps choose between fast/cheap models vs slow/accurate models.
Cost Efficiency Analysis
What this shows: Quality score per dollar spent (higher = better value). Calculated as: (accuracy score × tokens generated) / estimated cost. Small models often have best cost efficiency for simple tasks.
Model Consistency (Reliability)
What this shows: Variance in scores across tests. Lower variance = more predictable/reliable. Higher = inconsistent performance. Consistency matters for production use - you want predictable results.
